# Дизайн-документ: «Трасса» (рабочее название)

> Живой документ — карта, а не закон. Меняем по ходу.
> База: проект **The Hunter** (Godot 4.6, портрет 720×1280, iPhone/Xogot).
> Разрабатываем **слоями**: каждый слой — отдельная играбельная версия.

---

## 1. Концепция

Аркадный бесконечный мото-раннер в портрете под один палец. Ты — мотоциклист
на трассе. Держишь палец — разгоняешься, ведёшь — объезжаешь поток машин.
Чем быстрее едешь, тем больше километров (очков), но тем меньше времени увернуться.
Врезался — авария, конец заезда.

**Ядро одним предложением:**
*Быстрее едешь — больше километров, но меньше времени на реакцию.*

**Фишка игры (слой v3):** камеры и ограничения скорости. Гнать выгодно, но
превысишь под камерой — штраф. Вот это отличает игру от обычного доджера.

---

## 2. Жанр / платформа / сессия

- Жанр: аркадный endless racer / dodger.
- Платформа: iPhone (Xogot), портрет 720×1280, один палец.
- Сессия: 1–3 минуты, петля «ещё разок».

---

## 3. Управление (один палец делает всё)

- **Касание и удержание = газ.** Пока палец на экране — скорость растёт.
- **Ведёшь пальцем = руль.** Мотоцикл едет за пальцем по горизонтали (как сейчас охотник).
- **Отпустил = торможение.** Скорость плавно падает до минимума. Мотоцикл не встаёт
  колом — поток всё равно идёт (иначе можно «застопориться» и обнулить риск).

Отложено: второй режим «тапнул в точку — переехал». Смешивать две схемы запутанно,
а drag для быстрого объезда отзывчивее. Вернёмся, если захочется.

---

## 4. Что переиспользуем из The Hunter (~80% каркаса готово)

| Сейчас | Станет |
|---|---|
| Охотник едет за пальцем | Мотоцикл рулит влево-вправо |
| Спавнер добычи | Спавнер транспорта |
| Поймал красную → game over | Врезался → авария → game over (уже есть) |
| Рост скорости падения | Рост скорости/лимита |
| Счёт за ловлю | Километры пути |
| Рекорд (ConfigFile) | Лучший пробег |
| Тряска при гибели | Тряска при аварии (уже есть) |

## 5. Системы

### 5.1 Скорость (v1)
- `speed` — текущая, км/ч. Старт 40, минимум 30, потолок v1 = 120 (позже выше).
- Удержание: `speed += accel * delta`, accel ≈ 25 км/ч в секунду; не выше потолка/лимита.
- Отпустил: `speed -= decel * delta`, decel ≈ 15 км/ч в секунду; не ниже минимума.
- Скролл мира и поток привязаны к `speed`: быстрее едешь — всё несётся быстрее.
  `screen_px_per_sec = speed * K`, старт K ≈ 6, крутим по ощущению.

### 5.2 Километры / очки
- `distance_m += (speed / 3.6) * delta` (км/ч → м/с делением на 3.6).
- Очки = пройденные метры. Рекомендация: показывать **метры** — числа крупнее и приятнее
  («1 240 м»). Можно км с десятой — на вкус.
- Рекорд = лучший пробег (переиспользуем сохранение через ConfigFile).

### 5.3 Транспорт (v2+)
Одна сцена `Vehicle`, тип через `@export`:
- **Легковая:** средний размер и скорость.
- **Фура:** большая, длинная, медленная — трудно обогнать, перекрывает полосу.
- **Мото:** маленькая, быстрая.
Тип задаёт спрайт, размер коллизии и собственную скорость. В v1 — один тип-заглушка.

### 5.4 Полосы (v2+)
- Геометрия полос — ДАННЫЕ в коде (`var lane_count` + ширина), НЕ сцены: её читают
  отрисовка дороги, спавнер и игрок — нужен один источник правды.
- Смена числа полос (3↔2↔4) — в «чистой зоне»: стоп-спавн → ждём пустоты → плавно
  меняем ширину → новый `lane_count` → спавн дальше. Так перестроение (v4) не нужно.
- Транспорт спавнится по центрам полос. Игрок рулит свободно в пределах асфальта.

### 5.5 Лимиты и камеры (v3 — фишка)
- `limit` — текущий лимит, км/ч. Прогрессия зонами: 40→50→60→70→80→90→100→110→130→150.
- Спидометр показывает скорость и лимит; превышение подсвечивается красным.
- Зоны: каждые N метров лимит меняется (трасса — выше, населённый пункт — ниже).
  Дорожный знак может предупреждать о смене (по желанию).
- **Камера:** периодически появляется придорожная камера, скроллится вниз как объект.
  Когда игрок её проезжает при `speed > limit` → штраф: вспышка, звук, −X метров к счёту.
- Решение игрока: гнать ради км, но притормозить перед камерой и в населёнке.

### 5.6 Сложность / прогрессия
- Растёт потолок скорости и лимит.
- Плотнее поток (короче интервал спавна).
- Больше полос → фуры → перестроения (v4) → населённые пункты и дорожные работы (v5).

### 5.7 Архитектура дороги (решено)
Два разных слоя, не путать:

- **Визуал асфальта (скролл)** — НЕ из сцен. Штатный бесконечный скролл Godot:
  `ParallaxBackground`/`Parallax2D`, повтор текстуры по высоте, скорость скролла
  привязана к `speed`. Один узел, бесшовная петля.
- **Логика/контент** (число полос, населённые пункты, дорожные работы, знаки,
  места камер) — вот тут участок = отдельная сцена (chunk). Менеджер держит
  2–3 чанка стопкой, ушедший вниз перерабатывает (пул), новый сверху выбирает
  по правилам зоны. Трафик по-прежнему спавнит `Main` (чанки = дорога, не машины).
- **Зона** (зависит от пробега) — объединяет лимит + плотность камер + выбор чанка.
- **Разнообразие декора (на будущее):** при переработке чанка скрипт перезаполняет его
  взвешенным рандомом на тип — регулярная инфраструктура (столбы, отбойник) по фикс.
  шагу; редкие пропсы (камень, дерево) по шансу (`randf() < p`). Альтернатива — пул
  готовых вариантов чанка, выбираемых случайно при переносе.

**Когда:** чанки — это v2+. В v1 дорога = один `ParallaxBackground` + текущий
спавнер. Чанки в v1 — преждевременная сложность.

## 6. Роадмап (каждый слой играбелен)

### v1 — Драйв (играбельная база) ← начинаем отсюда
Цель: почувствовать вождение.
- Рескин: мотоцикл + машина (заглушки или ассеты — решим при сборке).
- Газ по удержанию + руль; торможение при отпускании.
- Поток машин сверху, скорость привязана к `speed`; **все смертельны** (столкновение = авария = game over).
- Счёт = метры. Скроллящаяся дорога для ощущения скорости. Базовый спидометр (число скорости).
- Переделать из текущего: убрать «хорошо/плохо» у добычи; спавнить машины; счёт по дистанции;
  добавить фон-дорогу; в игрока — модель скорости.
- **Готово, когда:** едешь, газуешь/тормозишь, объезжаешь поток, видишь скорость и метры,
  авария завершает заезд, рестарт работает.

### v2 — Полосы и типы транспорта
- 2 полосы → 3 → 4 (расширение по пробегу). Легковые / фуры / мото с разными размерами и скоростями.
- **Готово, когда:** многополосная дорога с разным транспортом, дорога умеет расширяться.

### v3 — Лимиты и камеры (фишка игры)
- Спидометр + текущий лимит с подсветкой превышения. Зоны со сменой лимита по пробегу.
  Камеры, штрафующие за превышение.
- **Готово, когда:** есть лимит и работающие камеры со штрафом, лимит меняется по зонам.

### v4 — Живой траффик
- Машины перестраиваются: сначала мигает поворотник (~1 c), потом перестроение.
  Это конечный автомат поведения на каждой машине — самая сложная часть, честно.
- **Готово, когда:** машины меняют полосу с видимым сигналом, и это влияет на объезд.

### v5+ — Контент и лоск
- Населённые пункты (лимит падает, плотнее камеры). Дорожные работы (перекрытия, конусы).
- Настоящий арт, звук мотора с изменением тона от скорости, меню, таблица рекордов.
- Дальше: день/ночь, погода, миссии.

## 7. Узлы в Godot (ориентир, детализируем при сборке)

- `Main` — менеджер: скорость, дистанция, лимит, спавн, состояние.
- `Player` (мотоцикл) — модель скорости + руль.
- `Vehicle` — одна сцена, тип через `@export` (v2).
- `SpeedCamera` — придорожная камера (v3).
- Дорога — `ParallaxBackground` с зеркальным повтором (штатный бесконечный скролл Godot)
  либо пара повторяющихся спрайтов, двигаемых в `_process`.
- Полосы — массив X-координат в `Main`.

## 8. Открытые вопросы (решим по ходу)

- Арт: **решено — Kenney Racing Pack** (top-down: дороги, машины, мото, декор, CC0). Брать как есть, не полировать.
- Метры или км на экране; размер штрафа за камеру.
- Игрок рулит свободно или «прилипает» к полосам (склоняюсь к свободному).
- **Попутный поток (обгон) vs встречный.** В дизайне выбран **попутный** (трасса, обгоняешь
  медленных) — лучше ложится на лимиты и обгоны. Можно пересмотреть.
- Минимальная скорость > 0, чтобы нельзя было «застопориться».

## 9. Чего НЕ делаем в v1 (защита от утопания)

Без полос, без типов транспорта, без лимитов/камер, без ИИ-перестроений, без населённых
пунктов. Только: едешь, газ/тормоз, объезжаешь поток, метры, авария = конец.
Всё остальное — следующие слои.
